# 真实性闸门（paper-verify）

润色和去 AI 味让文字更好看，但不保证内容为真。本 skill 专门检查**事实层面的硬伤**——这是论文被拒、被质疑甚至撤稿的主要原因，也是纯文风类 prompt 覆盖不到的盲区。

**核心原则：本 skill 只核查、只标记，不为了"看起来完整"而编造任何引用、数字或结论。核不准的，明确标为待核，交还给人。**

---

## 三个核查维度

### 维度一：引用真实性（Citations）

对正文中的每一处 `\cite{key}` / `[作者, 年份]` / 参考文献条目：

1. **存在性**：该文献是否真实存在？能否在可信来源（如有联网/检索能力则核实；否则基于已知知识判断）确认其标题、作者、发表年份、会议/期刊。
2. **匹配性**：引用所支撑的论断，与被引文献的实际内容是否相符？警惕"张冠李戴"——引了真实存在的文献，但它根本没说正文所宣称的内容。
3. **bib 一致性**：若同时提供 `.bib`，核对 `\cite` 的 key 是否都在 bib 中、bib 条目字段是否完整（缺 year/venue 等需标出）。

**处理规则（严禁幻觉补全）**：
- 能确认为真且匹配 → 标 `[OK]`。
- 无法确认真实性，或疑似 AI 生成的虚构文献（作者/标题/年份拼不出真实记录）→ 标 `[CITATION NEEDED]`，并说明疑点，**绝不替用户"补全"一个看似合理的引用**。
- 文献真实但与论断不匹配 → 标 `[MISMATCH]`，说明该文献实际讲什么。

### 维度二：数字一致性（Numbers）

1. 正文中陈述的每个关键数字（准确率、提升幅度、参数量、数据集规模等），与对应的**实验表格 / 提供的原始数据**逐一对照。
2. 检查内部自洽：摘要里的数字 ↔ 实验章节的数字 ↔ 表格里的数字，三处是否一致。
3. 检查派生计算：声称"提升 X 个百分点""快 Y 倍"等，是否与原始数值相符（自己算一遍）。

**处理规则**：
- 对得上 → `[OK]`。
- 对不上 / 表里查无此数 / 算出来不符 → `[NUMBER MISMATCH]`，并列出"正文写的值 vs 数据中的值"。**不擅自把正文改成表里的值**，只标出供人决定（可能是表错也可能是正文错）。

### 维度三：过度宣称（Overclaim）

1. 引言/摘要里声称的贡献，在实验部分是否**真的被验证**？有没有"说了要解决 A，实验只测了 B"。
2. 警惕无数据支撑的强断言：`state-of-the-art`、`significantly outperforms`、`first to`、`novel`、`完全解决`、`显著优于` 等，是否有对应实验/统计支撑。
3. 是否回避了不利结果、只报喜不报忧。

**处理规则**：
- 有支撑 → `[OK]`。
- 无支撑或夸大 → `[OVERCLAIM]`，指出宣称与证据的差距，并给出更稳妥的表述建议（如 `significantly` → 去掉或补显著性检验；`first to` → 加限定范围）。

## 输入方式

- **贴正文**：直接核查所提供文本（被 `paper-workflow` 调用时即此模式，核查当前流水线中的稿件）。
- **给文件路径**：若用户给出 `.tex` / `.bib` / 数据文件路径，**用 Read 工具读取**后核查；引用核查强烈建议同时提供 `.bib`，数字核查强烈建议同时提供实验数据表或 results 文件，否则只能做有限的内部自洽检查（并在报告中声明覆盖范围受限）。

## 输出格式

```
# 真实性核查报告

## 一、引用 (Citations)
- [OK] \cite{vaswani2017} — Attention Is All You Need，与"自注意力机制"论断匹配。
- [CITATION NEEDED] \cite{li2023fastllm} — 未能确认该文献存在，作者/标题/年份无法对应到真实记录，疑似虚构，请人工核对。
- [MISMATCH] \cite{he2016resnet} — 文献真实，但其内容为残差网络，不支撑此处关于"数据增强"的论断。

## 二、数字 (Numbers)
- [OK] 摘要"准确率 92.1%"与表2一致。
- [NUMBER MISMATCH] 正文称"提升 5 个百分点"，但表3为 88.0→90.5，实际为 2.5 个百分点。

## 三、过度宣称 (Overclaim)
- [OVERCLAIM] 摘要称"first to solve X"，但实验仅在单一数据集验证，建议限定为"on DATASET, we are the first to..."或补充证据。

## 覆盖范围说明
（若未提供 .bib / 实验数据，在此声明哪些核查只能做内部自洽、未能外部验证。）
```

若三个维度均无问题，输出：`[核查通过] 引用、数字与主张均未发现明显硬伤（注意：受限于可核验的信息范围，建议作者对标红项仍做人工终核）。`

---

## 与总控的衔接

`paper-workflow` 在逻辑检查之后、审稿之前调用本 skill：逻辑检查管"读得通不通"，本 skill 管"是不是真的"。标出的 `[CITATION NEEDED]` / `[NUMBER MISMATCH]` / `[OVERCLAIM]` 全部汇入最终输出的「修改意见」，且属于**必须人工处理**的高优先级项——这些是不能靠自动改写消除的事实性风险。
